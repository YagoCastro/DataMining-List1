Questão 1¶

O que uma transformao de atributo? Cite um exemplo de como isto pode ajudar os mtodos de aprendizado.

**Resposta:**

O processo de transfomao dos atributos ou das variveis um processo que est ligado a limpeza dos dados dentro de um projeto de cincia de dados. E entende-se como aes que so realizadas nessa etapa: A preparao de dados uma etapa crucial no processo de cincia de dados, pois responsvel por organizar, limpar e transformar os dados brutos para que possam ser utilizados na modelagem e avaliao. Essa fase consome as sadas da fase anterior, como os objetivos do negcio e o plano do projeto. O principal objetivo garantir que os dados estejam consistentes, precisos e adequados para extrair insights e permitir uma boa performance dos modelos. [1]
Essa tcnica nos permite converter os dados brutos em um formato necessrio para que os prximos procedimentos de processamento e modelagem de dados possam ser realizados com eficincia. Tecnicamente. A transformao de dados altera a estrutura, o formato e o valor dos dados e os torna limpos e utilizveis para os prximos processos.
Pode-se citar as seguites processos que fazem parte da transformao: - Suavizao dos dados: Como por exemplo quando aplicamos em um conjunto de dados desbalanceado um tcnica de oversample onde os dados da classe minoritria so adicionados mais vezes at o ponto em que eu tenho um balaceamento igualitrio entre as duas classe (considerando uma classe alvo binria), temos nessa situao uma necessidade de suavizao por mais que pequena nos dados gerados para garantir um melhor treinamento. - Dados faltantes: Na prpria limpeza dos dados de forma inicial temos que identificar os dados ausentes/faltantes e para esses existe tcnicas de transformao para que esse valor saia dessa questo de informao faltante pois algo ruim para anlise e aplicao do modelo, para uma informao - Outliers: Tambm temos o exemplo dos outliers, dados que se distnciam muito - Construo de novas variveis: Pode-se transformar ou agregar a um conjunto de dados um novo atributo a partir de outros atributos existentes na base, por exemplo, se eu tenho a altura e o peso, peo obter o IMC

Questão 3¶

Qual o motivo pelo qual os dados podem no ser IID (independentes e identicamente distribudos) no exemplo de filtragem de spam de e-mail da aula?

**Resposta:**

Uma das premissas fundamentais do aprendizado de mquina a suposio de que os dados utilizados para treinar um modelo so independentes e identicamente distribudos (IID). Essa suposio fundamental para a aplicao de muitos algoritmos de aprendizado de mquina, pois permite que sejam feitas inferncias estatsticas precisas a partir dos dados. Quando dizemos que os dados so independentes, estamos afirmando que as observaes no esto relacionadas entre si.
- Dependncia temporal: Em muitos casos, os dados podem apresentar dependncias temporais, ou seja, a ocorrncia de uma observao pode influenciar a ocorrncia de observaes futuras. Isso comum em sries temporais, onde os dados so coletados ao longo do tempo e podem apresentar padres temporais. Os padres nos e-mails podem mudar com o tempo. - Contexto especfico dos usurios: Os e-mails podem refletir padres pessoais e comportamentos dos usurios, como o tipo de remetentes, frequncia de mensagens e tpicos comuns. - Dependncias Espaciais: Alm disso, os dados podem apresentar dependncias espaciais, ou seja, a ocorrncia de uma observao pode influenciar a ocorrncia de observaes prximas no espao.

Questão 4¶

Qual a diferena entre um conjunto de validao e um conjunto de teste?

**Resposta:**

Fonte: [4]

- Conjunto de Treino: Este conjunto de dados usado para treinar o modelo. Ele utilizado para ajustar os parmetros do modelo. Ele vai ensinar o modelo a reconhecer padres e relaes nos dados. - Conjunto de Validao: Usado durante o treinamento para ajustar os hiperparmetros (como a taxa de aprendizado, nmero de camadas, nmero de rvores em um modelo de ensemble, etc.). **Isso acontece porque mais seguro para o projeto realizar um validao/ajuste com dados diferentes daqueles que foram utilizados para moldar/treinar o modelo.** Permite que o modelo generalize bem para novos dados, evitando problemas como overfitting (modelo ajustado demais aos dados de treino) ou underfitting (modelo subajustado). -- Aps cada iterao de treinamento, o modelo avaliado com os dados de validao. -- A mtrica de desempenho (por exemplo, acurcia, F1-score, etc.) informa se o modelo est aprendendo ou se est comeando a superajustar (overfitting). - Conjunto de Teste: Usado apenas uma vez no final do processo de treinamento e validao para avaliar o desempenho real do modelo **em dados nunca vistos antes.** Medir a capacidade de generalizao do modelo em dados completamente novos e fornecer uma estimativa final de sua performance.
Resumo: - Dados de treino so vistos pelo modelo e usados para ajustar os parmetros. - Dados de validao ajudam a evitar o overfitting ajustando os hiperparmetros. - Dados de teste avaliam a generalizao e garantem que o modelo funcione bem em dados futuros.

Questão 5¶

Por que no podemos (normalmente) usar o erro de treinamento para selecionar um hiperparmetro?

**Resposta:**

O ajuste de hiperparmetros seleciona hiperparmetros que daro bom desempenho de generalizao. Normalmente, isso funciona estimando o desempenho de generalizao para diferentes escolhas de hiperparmetros (por exemplo, usando um conjunto de validao) e, em seguida, escolhendo o melhor. Mas, como acima, essa estimativa ser superotimista se os mesmos dados tiverem sido usados para treinamento. A consequncia que hiperparmetros subtimos sero escolhidos. Em particular, haver um vis em direo a modelos de alta capacidade que sero superajustados.[5] Logo, No podemos usar o erro de treinamento para selecionar hiperparmetros porque ele no reflete a capacidade do modelo de generalizar para novos dados. O erro de treinamento mede apenas quo bem o modelo se ajusta aos dados nos quais foi treinado, mas no informa sobre sua performance em dados que nunca viu antes. O objetivo do ajuste de hiperparmetros equilibrar a troca de vis-varincia. O vis a divergncia entre as previses de um modelo e a realidade. Modelos que so subajustados, ou mal ajustados, no conseguem discernir relaes fundamentais entre pontos de dados e so incapazes de tirar as concluses exigidas para um desempenho preciso. A varincia a sensibilidade de um modelo a novos dados. Um modelo confivel deve fornecer resultados consistentes ao migrar de seus dados de treinamento para outros conjuntos de dados. No entanto, modelos com altos nveis de varincia so complexos demais: eles so superajustados aos seus conjuntos de dados de treinamento originais e tm dificuldade para acomodar novos dados. [6] Modelos com baixo vis so precisos, enquanto modelos com baixa varincia so consistentes. Um bom ajuste de hiperparmetros otimiza para criar o melhor modelo para o trabalho e, ao mesmo tempo, maximizar a eficincia dos recursos computacionais durante o treinamento. [6]

Questão 6¶

Nave Bayes faz a suposio de que todos os recursos so condicionalmente independentes, dado o rtulo da classe. Por que essa suposio necessria e o que aconteceria sem ela?

**Resposta:**

O algoritmo Nave Bayes faz uma suposio bastante ingnua, mas til para simplificar o clculo de probabilidades. Ele assume que cada atributo de uma instncia (por exemplo, caractersticas de um objeto) influencia independentemente a classificao dessa instncia em uma determinada classe (por exemplo, categorias de objetos). A suposio de independncia condicional em Nave Bayes necessria para simplificar os clculos, tornando o modelo escalvel e eficiente. Sem essa suposio, o modelo exigiria mais dados, maior poder computacional e seria mais complexo de treinar e interpretar.

Questão 7¶

Por que o KNN considerado um mtodo no paramtrico e quais so as consequncias indesejveis do design no paramtrico do KNN?

**Resposta:**

O algoritmo k-nearest neighbors (KNN), ou k vizinhos mais prximos, um classificador supervisionado no paramtrico, que usa proximidade para fazer classificaes ou previses sobre o agrupamento de um ponto de dados individual. Ele no assume nenhuma hiptese explcita sobre a distribuio dos dados subjacentes. Isso significa que o modelo no realiza nenhum treinamento propriamente dito para ajustar parmetros antes de fazer previses. Em vez disso, ele armazena os dados de treinamento e utiliza a proximidade entre os pontos para fazer classificaes ou regresses. No escala bem: como o KNN um algoritmo preguioso, ele consome mais memria e armazenamento de dados em comparao com outros classificadores. Isso pode ser caro do ponto de vista de tempo e dinheiro. Sensibilidade ao rudo:O KNN pode ser influenciado negativamente por outliers ou rudo nos dados de treinamento, j que os pontos mais prximos so usados diretamente para classificar ou prever. Maldio da dimensionalidade: Em espaos de alta dimensionalidade, as distncias entre os pontos tornam-se menos significativas, e a performance do KNN pode ser drasticamente afetada. Isso ocorre porque os pontos comeam a parecer igualmente distantes uns dos outros. Isso s vezes tambm chamado de fenmeno de pico, onde, aps o algoritmo atingir o nmero ideal de funcionalidades, a adio de mais delas aumenta a quantidade de erros de classificao, especialmente quando o tamanho da amostra menor.

Questão 8¶

Para qualquer modelo paramtrico, como o aumento do nmero de exemplos de treinamento afeta as duas partes do custo-benefcio fundamental?

**Resposta:**

- O aumento do nmero de exemplos de treinamento reduz a varincia, pois o modelo tem mais dados para aprender a verdadeira distribuio, tornando-o menos dependente de rudos ou peculiaridades do conjunto de treinamento. - O aumento do nmero de exemplos de treinamento geralmente no tem impacto significativo no vis, pois o vis depende principalmente da escolha da arquitetura ou do tipo de modelo. Por exemplo, se o modelo linear, ele no conseguir capturar relaes no lineares, independentemente de quantos dados sejam adicionados.
Resumo: - O aumento do nmero de exemplos de treinamento reduz a varincia, levando a um modelo mais robusto e menos propenso a overfitting. - No entanto, o aumento dos dados no reduz o vis, a menos que a complexidade do modelo seja ajustada para melhorar sua capacidade de capturar a verdadeira relao nos dados.

Questão 9¶

Tanto o aprendizado supervisionado quanto os modelos de agrupamento recebem uma entrada x_i e produzem um rtulo y_i. Qual a principal diferena?

**Resposta:**

- A principal diferena entre as duas abordagens a classificao entre conjuntos de dados rotulados e no rotulados. O aprendizado supervisionado utiliza conjuntos de dados com rtulos para treinar algoritmos de classificao ou previso. Dados identificados como "treinamento" so adicionados, e o modelo ajusta de forma iterativa a maneira como avalia diferentes caractersticas at alcanar o resultado desejado. Modelos de aprendizado supervisionado so mais rigorosos em comparao ao mtodo de aprendizagem sem superviso. Contudo, demandam o envolvimento do usurio durante o processamento de dados para garantir que a rotulagem seja feita adequadamente. [8] - Por outro lado, modelos de aprendizado no supervisionado sempre operam sem a interveno humana. Eles detectam e atingem uma estrutura composta por dados no rotulados. A nica colaborao do usurio aqui a validao das variveis de sada. Por exemplo, quando algum compra um novo laptop on-line, um modelo de aprendizado no supervisionado entende que o usurio pertence a um grupo de clientes que compram uma srie de produtos relacionados em conjunto. No entanto, a funo do analista de dados garantir que o sistema de recomendao possa oferecer opes para bolsas de laptop, protetores de tela e carregadores de carro. [8]

Questão 10¶

No agrupamento k-means, os clusters so garantidamente regies convexas. As reas que recebem o mesmo rtulo por KNN tambm so convexas?

**Resposta:**

- K-Means: No K-means, os clusters so formados minimizando a soma das distncias quadradas entre os pontos e o centride de cada cluster. Essa abordagem leva a regies convexas, pois a minimizao da distncia euclidiana resulta em divisores lineares (hiperplanos) entre os clusters. - K-Nearest Neighbors (KNN): No KNN, um ponto recebe o rtulo com base nos K vizinhos mais prximos no espao. As regies associadas a um rtulo dependem da distribuio dos pontos no conjunto de treinamento. As fronteiras entre os rtulos no so lineares nem necessariamente simples. Elas podem ter formas complexas e no convexas, j que a deciso baseada em uma votao local dos vizinhos mais prximos, que depende da posio dos pontos no conjunto de treinamento. As regies podem ter formas muito complexas, devido influncia local de cada ponto de treinamento.
